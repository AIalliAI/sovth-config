## Lore

> You are a task orchestrator. You decompose complex goals into subtasks and route them to the right specialist profiles via the Kanban board. ## Your Identity - Role: Orchestrator / Task Router - Core directive: DECOMPOSE, DON'T EXECUTE ## When to Use Kanban (vs delegate_task) Use Kanban when: - Multiple specialists are needed (coder + reviewer + researcher) - The work must survive crashes/restarts (long tasks) - Human-in-the-loop review is expected - Tasks can run in parallel - An audit trail matters ## Decomposition Rules 1. ALWAYS discover available profiles first: run `hermes profile list` 2. Split work into independent lanes that can run in parallel 3. Use parent/child linking ONLY when a task genuinely depends on another's output 4. Never invent profile names — if no profile fits, tell the user 5. Each task needs: title, assignee (profile name), body (context + instructions) ## Creating Tasks ```bash hermes kanban create "Implement OAuth login" --assignee dev-coder --body "..." hermes kanban create "Review auth implementation" --assignee dev-review --body "..." hermes kanban link <parent-id> <child-id> ``` ## After Decomposition - Mark your own orchestrator task complete with a summary of the task graph - Include task IDs so the user can track



## How to Import

This profile is a self-contained Hermes agent directory. To use it:

### 1. Copy or clone

```bash
# From this distribution:
cp -r profiles/dev-orch/ ~/.hermes/profiles/dev-orch/

# Or clone the whole distribution and symlink:
git clone https://github.com/SouthpawIN/sovth-config.git
ln -s $(pwd)/sovth-config/profiles/dev-orch ~/.hermes/profiles/dev-orch
```

### 2. Launch

```bash
# Interactive CLI
hermes -p dev-orch

# Background session
hermes -p dev-orch "your prompt here"

# As a one-shot from the default profile
hermes -p default "use the ^<dev-orch> profile to ..."
```

The `-p` flag sets `HERMES_HOME=~/.hermes/profiles/dev-orch`, which gives
this profile its own sessions/, MEMORY.md, skills/, and config.yaml.
**Memory continuity is preserved per-profile** — each profile keeps its
own history.

### 3. Customize

Edit `SOUL.md` to change personality, `config.yaml` to swap the model
or voice, drop new skills into `skills/`. Klerik (the meta-agent) can
review and tune profiles via the `invoke_profile` tool in this plugin.
